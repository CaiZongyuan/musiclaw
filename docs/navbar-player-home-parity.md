# Navbar / Player / 首页 Parity 清单

本清单用于对照 `/root/Projects/Frontend/YesPlayMusic` 的以下源码：

- `src/components/Navbar.vue`
- `src/components/Player.vue`
- `src/views/home.vue`

目标不是写出“风格相近”的新 UI，而是把这三块作为第一轮高优先级复刻对象进行收口。

## 1. Navbar

### 旧版基线

- 固定在页面顶部，半透明毛玻璃背景
- 左侧是后退 / 前进按钮
- 中间是 `Home / Explore / Library` 三个主入口
- 右侧是搜索框 + 用户头像
- 搜索框支持直接跳到搜索页
- 头像点击后进入用户相关操作

### 本轮已收口

- 当前全局壳体已从左侧栏改回顶部固定导航
- 主导航入口已调整为 `Home / Explore / Library`
- 搜索框已移回顶栏右侧，并支持回车进入 `/search`
- 头像入口已恢复在右上角
- 已补回 `/explore` 路由入口，避免导航落空
- 搜索框焦点态继续向旧版微调，并补回头像菜单的下拉交互。

### 仍未对齐

- 顶栏文案、按钮尺寸和 hover 细节仍需继续微调
- 旧版 Electron 自定义 titlebar 本轮未纳入范围

## 2. Player

### 旧版基线

- 底部常驻播放器
- 顶部是细进度条
- 中间三段式布局：
  - 左侧：封面、歌曲名、歌手、专辑来源/列表来源
  - 中间：上一首 / 播放暂停 / 下一首
  - 右侧：队列、循环、随机、音量、歌词入口
- 歌词是由底部播放器触发的全局层

### 本轮已收口

- 当前播放器已从纵向信息块改为更接近旧版的三段式控制条
- 进度条已移到播放器顶部，控制区顺序向旧版靠拢
- 左侧已恢复封面、歌曲名、歌手、专辑信息
- 右侧已补回“队列页”按钮，并补回 `/next` 路由入口
- 歌词入口继续保留在播放器右侧
- `Next Up` 已补成三段式结构：当前播放、插队播放、后续队列。

### 仍未对齐

- 喜欢按钮尚未恢复
- 队列页样式、图标细节和按钮尺寸仍需继续贴近旧版
- 歌词层目前仍是第一轮实现，不是最终视觉
- 旧版的列表来源跳转与更多细节动作仍未恢复

## 3. 首页

### 旧版基线

旧版首页区块顺序：

1. `by Apple Music`
2. `推荐歌单`
3. `For You`
4. `推荐歌手`
5. `新专辑`
6. `排行榜`

特点：

- 纵向内容流，不是仪表盘式两列面板
- 大标题 + 右侧 `See more`
- `CoverRow` 风格卡片，强调封面、标题、副文案
- `For You` 为双卡片结构

### 本轮已收口

- 首页已从仪表盘式 2x2 面板改回纵向内容流
- 区块顺序已按旧版重排
- `推荐歌单` / `推荐歌手` / `新专辑` / `排行榜` 已改为封面卡片流
- `For You` 已恢复双卡片结构
- 已补回首页依赖的 `/new-album`、`/daily/songs`、`/explore` 跳转入口
- `by Apple Music` 已改回旧版静态卡片数据来源。

### 仍未对齐

- `For You` 还未接回真实 `DailyTracksCard` / `FMCard` 业务能力
- 卡片间距、封面圆角、字体和 hover 细节仍需继续微调
- 首页各行的数据量、截断规则和副文案还需继续贴近旧版

## 4. 下一轮优先级

1. 继续微调 `Navbar` 的尺寸、字重、搜索框和头像交互
2. 继续微调 `Player` 的按钮尺寸、状态样式和队列页内容
3. 把首页卡片尺寸、封面比例、标题字重和 `For You` 真实业务接回去
4. 补旧版缺失的 `artist/:id/mv`、`library/liked-songs` 等路径
